Полное руководство по HTML для начинающих

Уровень 1: Фундамент (Самые основы)

1. Что такое HTML и структура документа

HTML (HyperText Markup Language — язык гипертекстовой разметки) — это скелет любой веб-страницы. Если сравнивать сайт с домом, то HTML — это фундамент, стены и перекрытия. Картинки, цвета и кнопки на места расставляют другие инструменты (CSS и JavaScript), но база создается именно здесь.

Главные понятия: теги и контейнеры. Всё в HTML строится на тегах. Тег — это команда для браузера, заключенная в угловые скобки: <название_тега>. Обычно теги парные: <тег> — открывающий (начало действия), </тег> — закрывающий (конец действия).

<p>Это абзац текста</p>

Базовая структура любой HTML-страницы: каждый HTML-документ должен иметь обязательный каркас.

<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Моя первая страница</title>
</head>
<body>
Здесь будет видимая часть сайта.
</body>
</html>

<!DOCTYPE html> — служебная инструкция. Она говорит браузеру: используй самый современный стандарт HTML, не включай древний режим совместимости. <html> — корневой тег, внутри которого находится вся страница. Атрибут lang="ru" указывает, что язык страницы — русский. <head> — голова страницы. Здесь хранится служебная информация, которая не видна на экране (кроме заголовка вкладки). <meta charset="UTF-8"> — тег мета-информации. charset="UTF-8" говорит браузеру, что текст нужно отображать в кодировке Unicode, чтобы не было кракозябр вместо букв. <title> — то, что вы видите на вкладке браузера. Очень важен для поисковиков. <body> — тело страницы. Всё, что написано здесь, пользователь видит на экране.

2. Основные теги для текста

Интернет на 90% состоит из текста, поэтому эти теги вы будете использовать чаще всего. Заголовки (<h1> — <h6>): заголовки нужны, чтобы выделять разделы. У них есть своя иерархия важности, как в книге.

  • <h1> — Самый главный заголовок страницы. Он должен быть один (как название книги).
  • <h2> — Заголовки разделов (как названия глав).
  • <h3> — Подразделы внутри глав.
  • И так до <h6>.
<h1>Это главный заголовок страницы</h1>
<h2>Это большой раздел</h2>
<h3>Это подтема</h3>

Параграфы (<p>): оборачивайте каждый абзац текста в тег <p>. Браузер автоматически добавит отступы сверху и снизу.

<p>Это первый абзац. Здесь много текста, и он отделен от другого абзаца пустой строкой (отступом).</p>
<p>Это второй абзац. Текст не сливается в одну сплошную стену.</p>

Форматирование текста: иногда нужно выделить слово жирным или курсивом. Для этого есть логические теги (они не просто меняют внешний вид, но и придают смысл):

  • <strong> — важность (жирный текст).
  • <em> — смысловое ударение (курсив).
<p>Это <strong>очень важное</strong> предупреждение. И <em>именно это</em> нужно запомнить.</p>

3. Списки

Списки бывают маркированные (буллиты) и нумерованные. Маркированный список (<ul> — Unordered List) используется, когда порядок элементов не важен (например, список продуктов).

<ul>
<li>Молоко</li>
<li>Хлеб</li>
<li>Яблоки</li>
</ul>

Результат: Молоко, Хлеб, Яблоки (каждый с новой строки с маркером).

Нумерованный список (<ol> — Ordered List) используется, когда важен порядок (например, инструкция или рейтинг).

<ol>
<li>Включить компьютер</li>
<li>Открыть браузер</li>
<li>Зайти на сайт</li>
</ol>

Результат: 1. Включить компьютер, 2. Открыть браузер, 3. Зайти на сайт.

Вложенные списки: внутри пункта списка (<li>) можно создать еще один список. Так делают меню или сложные структуры.

<ul>
<li>Пункт 1</li>
<li>Пункт 2
<ul>
<li>Подпункт 2.1</li>
<li>Подпункт 2.2</li>
</ul>
</li>
<li>Пункт 3</li>
</ul>

Уровень 2: Связи и адресация

4. Ссылки (<a>)

Ссылки — это то, что делает интернет паутиной. Тег ссылки называется a (от слова anchor — якорь). Главный атрибут: href. В нем указывается адрес, куда ведет ссылка.

<a href="https://www.google.com">Перейти на Google</a>

Относительные и абсолютные пути:

  • Абсолютные: полный адрес, начинается с http:// или https://. Ведет на другой сайт.
  • Относительные: путь внутри вашего сайта. Например, href="page2.html" или href="images/cat.jpg".

Открытие в новой вкладке (target="_blank"): по умолчанию ссылка открывается в той же вкладке. Чтобы открыть в новой, добавьте атрибут target:

<a href="https://yandex.ru" target="_blank">Яндекс (откроется в новом окне)</a>

Якоря (ссылки внутри страницы): если страница очень длинная, можно сделать навигацию по её разделам. Сначала ставим якорь — место, куда прыгать. Для этого нужен атрибут id (уникальный идентификатор).

<h2 id="chapter5">Глава 5</h2>

Затем делаем ссылку на этот якорь, используя символ решетки #.

<a href="#chapter5">Перейти к Главе 5</a>

5. Изображения (<img>)

Тег <img> уникален тем, что он одиночный (не требует закрывающего тега). Он сам вставляет картинку в то место, где написан. Обязательные атрибуты:

  • src (source) — путь к файлу картинки.
  • alt (alternative text) — текстовое описание. Оно покажется, если картинка не загрузится, и очень важно для незрячих людей (читалки озвучивают этот текст).
<img src="images/kotik.jpg" alt="Рыжий кот лежит на диване">

Размеры (необязательно, но полезно): можно задать ширину и высоту в пикселях. Если указать только ширину, высота подстроится автоматически, чтобы не нарушить пропорции.

<img src="images/kotik.jpg" alt="Кот" width="300">

Уровень 3: Продвинутая структура (DOM и компоновка)

6. Структура DOM

DOM (Document Object Model) — это модель документа. Представьте, что вы разобрали HTML-код на дерево, где каждая ветка — это тег, а маленькие веточки — теги внутри него.

Родители и дети: тег, внутри которого находятся другие — родитель. Теги, лежащие внутри родителя — дети. Теги внутри детей — внуки (или потомки).

<div> <!-- Это родитель -->
<p> <!-- Это ребенок div, и родитель для span -->
Текст <span>внутри</span> параграфа. <!-- span — внук для div -->
</p>
</div>

Понимание этой иерархии критически важно, когда мы дойдем до CSS и JavaScript, чтобы обращаться к конкретным элементам.

Блочные vs Строчные элементы: это деление очень важно для верстки.

  • Блочные (<div>, <h1>, <p>): занимают всю доступную ширину. Каждый новый блочный элемент начинается с новой строки. Как кирпичи, лежащие друг на друге.
  • Строчные (<span>, <a>, <strong>): занимают только то место, которое нужно их содержимому. Они располагаются в строку друг за другом, как слова в предложении. Перенос на новую строку происходит, только если не хватает места.

<div> и <span> — это универсальные контейнеры (пустые коробки), которые нужны для группировки элементов, чтобы потом их стилизовать или двигать.

7. Семантическая верстка

Раньше люди верстали сайты одними <div>-ами. Это работало, но компьютеру было сложно понять, где шапка, а где подвал. Семантические теги решают эту проблему.

  • <header> — Шапка сайта или раздела. Обычно там логотип и меню.
  • <nav> — Блок навигации (меню со ссылками).
  • <main> — Главное уникальное содержимое страницы (то, что не повторяется на других страницах).
  • <article> — Независимая, целостная часть сайта (например, новость, пост в блоге).
  • <section> — Крупный раздел (например, глава в статье).
  • <aside> — Боковая колонка (сайдбар), содержимое косвенно связанное с главным.
  • <footer> — Подвал сайта. Обычно там копирайт, контакты.

Зачем это нужно?

  • Для SEO (поисковиков): поисковик лучше понимает структуру сайта.
  • Для доступности: программы экранного чтения могут быстро перейти к главному контенту (<main>), пропустив меню.

Уровень 4: Работа с данными (Таблицы)

8. Создание таблиц

Таблицы в HTML создаются рядами (строками). Сначала вы говорите начало строки, потом кладете в нее ячейки.

  • <table> — контейнер таблицы.
  • <tr> (table row) — строка.
  • <td> (table data) — обычная ячейка.
  • <th> (table header) — ячейка-заголовок (текст в ней жирный и по центру).

Простая таблица (2x2):

<table border="1"> <!-- border="1" добавлен для наглядности, обычно рисуют CSS -->
<tr>
<th>Имя</th>
<th>Возраст</th>
</tr>
<tr>
<td>Анна</td>
<td>25</td>
</tr>
<tr>
<td>Иван</td>
<td>30</td>
</tr>
</table>

Структурные теги: для больших таблиц используют логические блоки:

  • <thead> — для шапки.
  • <tbody> — для основной массы данных.
  • <tfoot> — для итогов (внизу).

9. Объединение ячеек

Иногда ячейка должна занимать место двух или трех. Это называется объединение.

  • colspan — объединение по горизонтали (сколько столбцов занимает ячейка).
  • rowspan — объединение по вертикали (сколько строк занимает ячейка).
<table border="1">
<tr>
<th>ФИО</th>
<th colspan="2">Телефоны</th> <!-- Эта ячейка займет 2 колонки -->
</tr>
<tr>
<td>Петров П.П.</td>
<td>8911...</td>
<td>8912...</td>
</tr>
</table>

Результат: В первой строке будет всего две ячейки: "ФИО" и большая ячейка "Телефоны", которая растянута на два столбца.

Уровень 5: Интерактив (Формы)

10. Основы форм (<form>)

Формы — это способ отправить данные с компьютера пользователя на сервер. Это лайки, комментарии, регистрация, заказ товара. Тег <form> — это контейнер. Сам по себе он не виден, но он говорит браузеру: данные из полей внутри меня нужно собрать и отправить.

<form action="/submit-form" method="post">
<!-- Здесь будут поля ввода -->
</form>
  • action — адрес программы на сервере, которая будет обрабатывать данные.
  • method — способ отправки (get или post). Для простоты запомните: post — для паролей и больших данных, get — для поиска (данные будут видны в адресной строке).

11. Поля ввода (<input>)

Это самый популярный тег внутри формы. Он универсален и меняет свой вид в зависимости от атрибута type.

  • type="text" — обычное поле для ввода строки.
<input type="text" name="username">
  • type="password" — поле для пароля (точки вместо букв).
<input type="password" name="userpass">
  • type="email" — поле для почты (мобильные браузеры подставят клавиатуру с @).
  • type="number" — поле для чисел (со стрелочками +/-, нельзя ввести буквы).
  • type="checkbox" — квадратик для галочки (выбор нескольких вариантов).
<input type="checkbox" name="hobby" value="music"> Музыка
<input type="checkbox" name="hobby" value="sport"> Спорт
  • type="radio" — кружочки для выбора одного варианта из нескольких (одинаковый name).
<input type="radio" name="gender" value="male"> Мужской
<input type="radio" name="gender" value="female"> Женский
  • type="file" — кнопка для загрузки файла.

12. Текстовые поля и списки

<textarea> — многострочный текст. Используется для комментариев, отзывов. В отличие от <input>, это парный тег.

<textarea name="comment" rows="5" cols="40">Текст по умолчанию</textarea>

rows — высота в строках, cols — ширина в символах.

<select> — выпадающий список. Экономит место на экране.

<select name="city">
<option value="msk">Москва</option>
<option value="spb">Санкт-Петербург</option>
<option value="kaz" selected>Казань</option> <!-- selected - выбран по умолчанию -->
</select>

13. Кнопки и отправка

  • <button> — более гибкая кнопка (внутрь можно положить картинку или жирный текст).
  • <input type="submit"> — классическая кнопка отправки.
<button type="submit">Отправить данные</button>

или

<input type="submit" value="Отправить">

Группировка полей: <fieldset> рисует рамку вокруг группы полей, а <legend> — заголовок этой группы.

<fieldset>
<legend>Контактные данные</legend>
Имя: <input type="text" name="name"><br>
Email: <input type="email" name="email">
</fieldset>

Уровень 6: Атрибуты — детали деталей

14. Глобальные атрибуты

Эти атрибуты можно добавить к любому HTML тегу.

  • id — Уникальное имя элемента. На одной странице не может быть двух элементов с одинаковым id. Нужен для якорей, CSS и JavaScript.
<div id="header"></div>
  • class — Имя класса. В отличие от id, классы могут повторяться. Нужен в основном для CSS, чтобы стилизовать группу элементов.
<div class="product-card"></div>
<div class="product-card"></div> <!-- Можно повторять -->
  • title — Всплывающая подсказка при наведении мыши.
<p title="Это подсказка">Наведи на меня мышь</p>
  • style — Позволяет написать CSS-стили прямо внутри тега (обычно так не делают, но для экспериментов пойдет).
<p style="color: blue; font-size: 20px;">Синий текст</p>

15. Специфические атрибуты (в основном для форм)

  • placeholder — Полупрозрачная подсказка внутри поля, которая исчезает, когда начинаешь печатать.
<input type="text" placeholder="Введите имя">
  • required — Поле обязательно для заполнения. Форму нельзя будет отправить, пока поле не заполнено.
<input type="text" required>
  • disabled — Поле заблокировано (нельзя кликнуть, изменить, и данные с него не отправятся).
<input type="text" value="Неактивно" disabled>
  • readonly — Поле только для чтения (посмотреть можно, изменить — нельзя, но данные отправятся).
<input type="text" value="Только чтение" readonly>

Бонус-трек: Что дальше?

Поздравляем! Вы изучили базовый синтаксис и основные теги. Но HTML редко используется сам по себе.

  • CSS (Cascading Style Sheets): чтобы раскрасить этот "скелет", сделать красивые отступы, тени, анимацию и расположить всё красиво на странице. Подключается тегом <link> внутри <head>.
  • JavaScript (JS): чтобы страница ожила: реагировала на клики, показывала всплывающие окна, подгружала данные без перезагрузки страницы. Подключается тегом <script>.

Этот фундамент — ваша база. С ней можно смело переходить к изучению CSS и верстке макетов. Удачи!
